# AegisMesh：把“订阅转换”做成一条安全闭环流程 ✨

> 这不是又一个“节点工具”，而是一个**防泄露的订阅中转工作流**。  
> 目标很简单：你可以用外部转换能力，但不把真实节点裸奔给外站。

---

## 01｜我为什么做这个工具？🤔

很多人都在用在线订阅转换（比如 OpenClash 流程）：

1. 粘贴订阅链接  
2. 站点转换  
3. 下载配置

问题是：  
真实 `server/ip + port + 部分敏感字段` 会在中间环节暴露。  

**AegisMesh 的核心想法**：  
先把真实节点“伪装”成假 endpoint，再去外部转换；  
最后拿回结果，按本地映射“还原”。

一句话：**能力借外站，隐私留本地**。

---

## 02｜产品怎么设计成“可用”而不是“能跑”🧩

我把流程拆成两条互不干扰的模式（按钮切换）：

- 网站转换流程（文本粘贴式）
- OpenClash 链接流程（完整 YAML 中转式）

这样做的原因：

- 老用户习惯不变
- 新流程专门适配 OpenClash 在线转换
- 出问题时定位更快，不会互相污染状态

---

## 03｜OpenClash 模式到底怎么跑？🚀

### Step A：抓取
- 输入原订阅 URL
- 用 `clash.meta / clash / mihomo` User-Agent 拉取
- 必须拿到完整 `Clash YAML`（不是散节点）

### Step B：伪装
- 本地解析 `proxies`
- 把真实 endpoint 改成假域名/假 IP + 假端口
- 生成本地映射（profile 级别）

### Step C：上传
- 伪装后的完整 YAML 上传 OpenList
- 一键复制假链接 + 二维码扫码

### Step D：还原
- 贴回 OpenClash 转换结果
- 本地按映射还原真实 endpoint
- 输出 YAML，保证客户端可直接用

---

## 03.5｜手把手教程（可直接照做）🧭

### 教程 A：OpenClash 链接模式（推荐）

1. 打开 AegisMesh，先点 `OpenList设置`。  
2. 填 `base_url / 用户名 / 密码 / 目标目录`，点 `联通测试` 成功后保存。  
3. 切到 `OpenClash链接流程`，粘贴订阅 URL。  
4. 点 `抓取并伪装上传`，拿到伪装链接。  
5. 点 `复制伪装后链接到剪贴板`，把链接给 OpenClash 在线转换。  
6. 把 OpenClash 返回内容粘回步骤③，点 `执行还原`。  
7. 步骤④点 `保存还原文档`，生成 `.yaml` 导入客户端。  

### 教程 B：网站转换模式（经典）

1. 切到 `网站转换流程`，粘贴订阅文本（或从 URL 抓取）。  
2. 点 `提取并展示节点`，再点 `执行伪装`。  
3. 复制伪装后内容，提交到转换网站。  
4. 把转换结果粘回软件，点 `执行还原`。  
5. 保存 YAML，导入客户端。

## 04｜实现重点（技术层）⚙️

### 1) 三重匹配还原（容错核心）
还原时不只看一种线索，而是：

- `NID`（若存在）
- 假 endpoint 命中
- 真 endpoint 命中（幂等还原，解决“外站已回填真实地址”）

这一步是稳定性的关键。

### 2) OpenList 上传兼容
为了解决“某些环境上传失败”，做了多重兜底：

- 认证头同时兼容 `token` / `Bearer token`
- 优先 `form` 上传，失败回退 `put` 上传
- 上传前自动 `mkdir` 远程目录
- SSL 校验失败自动无验证重试（按你的使用场景）
- 错误信息带 HTTP 与服务端返回，方便排障

### 3) 输出规范化
- 还原保存统一输出 `.yaml`
- 文件名毫秒级时间戳（无特殊字符）
- 保存后按保留天数清理旧文件

## 05｜做这个项目时踩过的坑 🧯

- 转换站会改名字、改顺序、改格式，导致“严格一把梭”必炸
- 订阅地址返回内容依赖 User-Agent，不同 UA 返回结构完全不同
- OpenList 在不同版本/代理链下，上传接口行为会不一致
- 只处理“干净节点”不够，必须处理“完整配置里的分组与规则引用”

- 双流程并行：网站转换 + OpenClash
